
# Smart Rover Speed & Direction Control System using ANFIS

A production-grade intelligent control project implementing an **Adaptive Neuro-Fuzzy Inference System (ANFIS)** to govern the navigational velocity and turning angles of an autonomous Rover. The framework synthesizes the structured, human-like reasoning of **Fuzzy Logic (Mamdani/Sugeno)** with the backpropagation data-driven optimization of **Deep Neural Networks (PyTorch)** to resolve complex, non-linear trajectory adjustments.

This repository provides the complete pipeline: from baseline Fuzzy Inference System (FIS) simulation and combinatorial dataset generation, to hybrid XANFIS regression modeling and empirical metrics evaluation.

---

## 🎯 1. Problem Statement & Objective

Autonomous robotic navigation in unmapped or dynamic environments demands high-precision, real-time control loops. Traditional mathematical control models often struggle with non-linear environmental variables, while pure deep learning models operate as uninterpretable "black boxes."

**Project Objectives:**
* **Knowledge Representation:** Construct a baseline Fuzzy Logic controller to map a Rover's positional inputs (`Distance`, `Angle`) into navigational outputs (`Speed`, `Turn`).
* **Hybrid Optimization:** Employ a 5-layer **Sugeno-type ANFIS** architecture to fine-tune membership function parameters through gradient descent, reducing empirical error while retaining system interpretability.
* **High-Fidelity Regression:** Achieve exceptional generalization on unseen situational coordinates without architectural overfitting.

---

## 🏗️ 2. Methodology & Architectural Design

The controller is designed as a hybrid system divided into two core operational phases:

### Phase I: Baseline Mamdani Fuzzy Controller
Developed using `scikit-fuzzy` to establish core heuristic navigation principles:
* **Inputs (Antecedents):** * `Distance` (Range: $[0, 100]$): Classified into `Close`, `Medium`, and `Far`.
  * `Angle` (Range: $[-90, 90]$): Classified into `Right`, `Straight`, and `Left`.
* **Outputs (Consequents):**
  * `Speed` (Range: $[0, 100]$): Universal velocity mapped as `Slow`, `Medium`, and `Fast`.
  * `Turn` (Range: $[-90, 90]$): Heading adjustments mapped as `Right`, `Straight`, and `Left`.

### Phase II: 5-Layer Sugeno-Type XANFIS Network
Built on top of PyTorch via the `xanfis` library, parsing the fuzzy components into a fully differentiable neural architecture trained via Backpropagation:
1. **Layer 1 (Fuzzification):** Passes inputs through learned Triangular Membership Functions (`trimf`).
2. **Layer 2 (Rule Firing):** Computes the firing strength of the 5 core behavioral rules using T-norm operations.
3. **Layer 3 (Normalization):** Normalizes individual rule strengths against global activations.
4. **Layer 4 (Consequent Layer):** Evaluates linear Sugeno polynomial functions for each rule zone.
5. **Layer 5 (Defuzzification):** Computes the final unified continuous regression vector ($Y$) via weighted linear aggregation.

## 📊 3. Dataset Generation & Preprocessing

To train the neuro-fuzzy pipeline, an exhaustive combinatorial dataset was synthetically generated by simulating all physical boundary intersections:
* **Data Scale:** Complete grid exploration traversing $101$ distance points and $181$ angular steps, yielding a total matrix of **18,281 empirical samples**.
* **Train/Test Splitting:** Partitioned using a rigorous **85% Training** ($15,538$ samples) and **15% Testing** ($2,743$ samples) distribution ratio to prevent training bias.
* **Feature Scaling:** Inputs and target outputs are structured into tensor matrices and bound via `MinMaxScaler` mapping directly to the optimization interval $[0, 1]$.
