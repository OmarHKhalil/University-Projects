
# 🚗 Car Showroom Website - Web Programming 1 (WEP 1) Project

An integrated, responsive web application designed for browsing and managing premium cars for sale from three world-leading manufacturers: **Ferrari**, **Mercedes-Benz**, and **BMW**. This project was developed as a core academic requirement for the Web Programming 1 course at Arab International University (AIU).

## 🎯 Main Components
The web ecosystem consists of a structured network of interconnected pages:
1. **Homepage:** The initial landing gateway featuring top deals, latest automotive news, and featured cars.
2. **Login Page:** A secure validation checkpoint for returning registered users.
3. **Sign-Up Page:** An onboarding portal allowing new guests to create dedicated profiles.
4. **Admin Page:** A secure backend control center reserved for showroom fleet management.
5. **Product Page:** The main catalog filtering cars into three premium categories (Ferrari, Mercedes, BMW).
6. **Terms & Conditions Page:** Explains user agreements, sales policies, and available purchasing facilities.

## ⚙️ User Workflow
1. Visitors land on the **Homepage** to skim general deals, promotional highlights, and car industry news.
2. If a guest attempts to view the detailed **Product Page** or initiate an ordering intent, the system enforces access control by prompting the **Login Page**.
3. New users are redirected to the **Sign-Up Page**, and upon successful database profile initialization, they return to the homepage to access full catalogs.
4. Authorized **Admins** bypass standard workflows to access the restricted management panel, where they can execute instant insertions, deletions, and pricing updates.
